    这篇文章主要是在讲：如何搭建并配置 3X-UI，并结合 Cloudflare 的 A 记录、证书和回源端口设置，最终完成 VLESS + XHTTP + TLS 的接入，再把订阅导入客户端进行测试。整体流程就是先安装面板，再绑定域名、配置证书和入站，最后通过 Cloudflare 把流量转发到 3X-UI 设置的端口。🚀 1、安装 3X-UI 使用文中的一条安装命令即可完成安装，安装过程中基本一路按 Enter 就行，操作比较简单。 2、在 Cloudflare 添加 A 记录 需要在 CF 里添加一条 A 记录： 类型是 A 名称是域名前缀 内容填写服务器 IP 代理要开启 这一步的作用是让域名先指向你的服务器，并通过 Cloudflare 代理。 3、获取域名证书的公钥和私钥 接下来需要准备域名证书里的公钥和私钥，后面在 3X-UI 的 TLS 配置里会用到。 简单理解，就是给你的连接加密用的身份凭证。🔐 4、配置 3X-UI 入站 入站设置里主要内容如下： 备注可以随便写 协议选择 vless 端口可以自定义 传输方式选择 XHTTP 这是文章里的核心协议配置部分。 5、设置 TLS 信息 在安全选项里选择 TLS SNI 填写你在 Cloudflare 配置的域名 公钥和私钥填写前面获取到的源服务器证书内容 这样就把域名、证书和 3X-UI 入站关联起来了。 6、配置 Cloudflare 回源端口 这一步是让 Cloudflare 知道访问这个域名时，应该把 HTTPS 请求转发到你 3X-UI 里设置的协议端口。 文章里提到要： 选择更改端口 设置规则条件，主机名等于你设置了 A 记录的域名 并且 HTTPS 条件成立 最后把端口改成 3X-UI 中设置的协议端口 这一步很关键，不然流量可能到不了面板对应的服务端口。⚙️ 7、导入订阅并测试 最后把订阅导入客户端测试是否可用。 文章还提供了几个优选域名、优选 IP 和 Cloudflare 节点参考地址，方便选择更稳定的入口。 可选的 CF HTTPS 回源端口包括： 443、8443、2053、2083、2087、2096 任选其一即可。 整体来看，这篇文章就是一套从安装 3X-UI 到通过 Cloudflare 完成域名接入、证书配置、端口回源和订阅测试的完整流程，适合按步骤照着配置。✅
