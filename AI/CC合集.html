    这篇文章主要是在整理一套 Claude Code 相关的使用方案，内容包括安装、配置、启动参数、提示词规则、代理加速、MCP 工具、技能扩展、中转站，以及请求头示例等。整体看下来，它更像是一份“Claude Code 增强工作流配置清单”🛠️，目的是让工具更顺手、减少干扰、接入更多搜索和自动化能力，同时适配 Windows 环境和中文使用习惯。 1. 安装部分 文章先列了需要准备的工具，包括 CC Switch、claude code、codex、gemini，以及 CCG、CCline。 还给了两个命令： npx ccg-workflow npx get-shit-done-cc@latest 这部分的重点是先把整套工作流相关工具装起来，打基础 🚀 2. claude.settings 配置 这是文章里最核心的内容之一。 主要做了几件事： 关闭一些不必要的提示和署名，比如 includeCoAuthoredBy、spinnerTipsEnabled 设置权限，只允许 Bash、Read、Edit 默认模式设为 acceptEdits 配置多个环境变量，用来减少非必要流量、关闭遥测、提高输出 token 上限等 把状态栏命令设成 ccline 在 SessionStart 时自动执行一个 node 脚本检查更新 简单说，这部分是在把 Claude Code 调整成“更安静、更高权限、更适合长期使用”的状态 ⚙️ 3. claude.json 免登录与去 Tips 这里只放了一条： "hasCompletedOnboarding": true 意思是跳过新手引导，并减少提示信息。作用很直接，就是让启动界面更简洁。 4. 启动参数 文章给出： claude --dangerously-skip-permissions 这表示启动时跳过权限确认。 好处是操作更快，但从名字也能看出来，这属于比较激进的方式，适合对环境很熟、明确知道自己在做什么的时候使用 ⚠️ 5. .md 提示词规范 这一段非常像“给 AI 的行为守则”，而且写得很细。 重点包括： 全程中文、用大白话、默认短回答 改代码前先分析，再给方案，等确认后再动手 需求不清时，最多问 3 个关键问题 只改必要部分，不整片重写 必要时提醒变量是否硬编码 改完必须自检，能跑测试就跑 多步骤任务才列 ToDo 当前环境是 Windows，不用不兼容 bash 命令 回答代码问题前，必须先用 contextweaver 定位代码 不确定时要联网搜索，优先 exa-proxy 扫描项目、找 bug、跑测试要用 agents 这部分的意义是让 AI 更像一个“稳妥的协作开发助手” 🤖 6. 加速代理 文中给了一个代理地址： https://betterclau.de/claude/ 作用就是给 Claude 相关请求提速，适合网络不稳定或者访问较慢的场景。 7. MCP 工具 这一部分是在扩展 Claude Code 的外部能力。 包括： exa-proxy：用于联网搜索、代码上下文获取、网页抓取、深度研究 agentation：网页标注 chrome_devtools_mcp：浏览器开发 Context7：浏览器搜索 Context Weaver：代码定位 还给了 contextweaver 的启动配置： command 是 contextweaver args 是 mcp 也就是说，这套配置希望 AI 不只是聊天，还能真正接入搜索、浏览器、代码定位等工具链 🔌 8. skills 扩展 文章还列了一些可选技能包，比如： 去 AI 味写作增强 网页 UI/UX 美化 浏览器开发 ralph-wiggum 内部插件 browserwing 浏览器自动化 其中 browserwing 还给了安装和启动方法： npm install -g browserwing browserwing --port 8080 这说明作者不只是想用 Claude 写代码，还想让它参与页面优化、浏览器控制和自动化任务 🌐 9. 中转站 这里列了多个网址，像是可选的服务入口或 API 中转平台。 作者还特别标了一句“听说很便宜”，说明这一部分更多是在收集可替代服务资源。 10. CC 请求头示例 这一节展示了 Claude Code 请求时的一组 HTTP 头信息。 里面包含： Anthropic-Beta Anthropic-Version User-Agent X-Stainless 系列字段 Accept: text/event-stream 这部分更偏底层，适合想研究 Claude CLI 请求结构、接口行为、或者做兼容接入的人看 🧩 11. 额外模型 最后还补充了一个本地模型命令： ollama run huihui_ai/qwen3.5-abliterated:9b 说明作者除了 Claude 生态，也关注本地模型替代方案，方便在某些场景下做补充使用。 整体来说，这篇文章的重点不是讲理论，而是在搭一套实用的 Claude Code 增强环境 🛠️。核心思路可以概括为：先装齐工具，再通过配置减少干扰、放宽权限、接入搜索和自动化能力，最后配上适合中文和 Windows 的提示词规则，让整套开发体验更顺手。
